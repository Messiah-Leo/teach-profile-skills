## Default Loop

Use or adapt this loop:

```text
domain constraints -> model -> method -> implementation
-> integration/use -> feedback metrics -> revision
```

## How To Use This In Lessons

When explaining a concept, ask how it appears in the loop:

- Constraints: What rules, requirements, boundaries, or assumptions matter?
- Model: What variables, concepts, structures, or representations are introduced?
- Method: What procedure, algorithm, experiment, practice, or intervention is used?
- Implementation: What tools, artifacts, APIs, processes, or tests matter?
- Integration: Where does this work meet the real environment or users?
- Feedback: What signals tell us whether to revise?

## Preferred Explanation Shape For Derivations

1. State the constraint or motivating problem.
2. Define the smallest necessary terms or variables.
3. Show a concrete toy case.
4. Generalize the structure.
5. Check boundary cases, units, assumptions, examples, or failure modes when relevant.
6. Connect the result to application, implementation, or verification.

## Useful Transfer Questions

- What is the system boundary?
- Which quantities are states, inputs, outputs, or feedback signals?
- What is directly observable, and what is inferred through a model?
- What feedback signal is meaningful?
- Is the current optimization local, or does it improve the full workflow?
- Where does the tool or environment behave like a black box?
