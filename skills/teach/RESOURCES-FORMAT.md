# RESOURCES.md Format

`RESOURCES.md` is the curated source list for the topic. Lessons should draw from these sources instead of unsupported guesses.

## Structure

```md
# {Topic} Resources

## Knowledge

- [Book or article title](https://example.com)
  What it covers. Use for: when to consult it.

## Wisdom (Communities)

- [Community name](https://example.com)
  Why it is useful. Use for: real-world feedback, critique, or practice.

## Gaps

- {Missing source area that still needs research}
```

## Rules

- Prefer primary sources, recognized experts, peer-reviewed work, official documentation, and well-moderated communities.
- Annotate every resource.
- Group sources by Knowledge and Wisdom.
- Record gaps explicitly.
- Remove sources that turn out to be weak or off-mission.
