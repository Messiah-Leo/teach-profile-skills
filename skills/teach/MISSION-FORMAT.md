# MISSION.md Format

`MISSION.md` lives at the workspace root. It captures why the user is learning this topic. Every teaching decision should trace back to it.

## Template

```md
# Mission: {Topic}

## Why
{1-3 sentences. The concrete real-world goal the user is chasing. What changes in their life or work when they learn this?}

## Success looks like
- {A specific, observable thing the user will be able to do}
- {Another specific thing}

## Constraints
- {Time, tools, prior commitments, learning preferences, or other boundaries}

## Out of scope
- {Adjacent topics that should not distract from the current goal}
```

## Rules

- One mission per workspace.
- Prefer concrete outcomes over abstract wishes.
- Push back on vague missions.
- Revise the mission when the goal changes.
- Keep the mission short enough to remain useful.
