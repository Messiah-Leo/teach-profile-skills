# Learning Record Format

Learning records live in `learning-records/` and use sequential numbering such as `0001-slug.md`.

## Template

```md
# {Short title}

{1-3 sentences explaining what was learned or established, and why it matters for future sessions.}
```

## When To Write One

Write a learning record when:

1. the user demonstrates genuine understanding,
2. the user discloses meaningful prior knowledge,
3. a misconception is corrected,
4. the mission shifts.

Do not write learning records for mere topic coverage or session logs.

## Optional Sections

Use only when helpful:

- `Status: active | superseded by LR-NNNN`
- `Evidence`
- `Implications`
