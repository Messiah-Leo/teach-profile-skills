# Customization

The public repository contains safe placeholders. Personalize a local copy
before using the profile skill seriously.

## Files To Edit

### `references/global-profile.md`

Use this for stable, cross-course learner context:

- durable learning goals,
- recurring work or study identity,
- preferred transfer contexts,
- broad teaching orientation.

Do not store temporary moods, private file paths, or one-off project details.

### `references/research-workflow.md`

Use this for the recurring workflow that should anchor abstract explanations.
For example:

- scientific research,
- mathematical derivation,
- software engineering,
- product or management work,
- creative practice.

Keep the workflow short enough that Codex can actually use it during a lesson.

### `references/cognition-and-planning.md`

Use this for stable planning and self-regulation preferences. Avoid diagnostic
labels or sensitive inferences unless the learner explicitly wants them stored.

## What Belongs In The Teach Workspace

Course-local material should stay with the current learning workspace:

- `MISSION.md`,
- `RESOURCES.md`,
- `NOTES.md`,
- `learning-records/*.md`,
- `lessons/*.html`,
- `reference/*.md` or `reference/*.html`,
- `GLOSSARY.md`.

If a fact only matters for one course, put it in the course workspace, not in
the global learner profile.

## Profile Update Rule

Do not update stable profile files after every lesson.

After five substantial lessons, use:

- `templates/five-lesson-review.md`,
- `templates/profile-update-proposal.md`,
- `references/update-protocol.md`.

Only update the profile when the pattern is repeated, explicitly confirmed, or
clearly useful across future courses.

## Privacy Checklist

Before publishing or sharing a customized copy, check for:

- names and contact details,
- private research or client information,
- local file paths,
- note-vault paths,
- API keys and tokens,
- sensitive personal inferences.
